1) What the model takes as INPUT
====================================================================

A) Data inputs
- Airports file: data/sample_airline/airports.csv
  Required fields: airport_id, iata, lat, lon (plus metadata like name).
- Routes file: data/sample_airline/routes.csv
  Required fields: source_airport_id, target_airport_id.
- These static files define topology (who is connected to whom) and coordinates.

B) Synthetic temporal signal inputs
- The repository creates time-varying edge weights from the static graph using:
  - snapshots (T)
  - seasonal_strength
  - noise_std
- In default configs this appears as synthetic traffic/activation probability over time.

C) Configuration inputs (main knobs)
From configs/airline_synth.yaml and configs/airline_pretrain_reusable.yaml:
- simulator:
  - mc_rollouts (Monte Carlo count for raw evaluation)
  - activation_threshold
  - coverage_radius_km
- model (diffusor / surrogate):
  - hidden_dim, layers
  - diffusor_lr, diffusor_epochs
  - teacher_mc_rollouts
  - marginal_loss_weight, ranking_loss_weight
- rl (policy training):
  - budget k (number of seeds selected)
  - gamma, lr
  - train_episodes, warmup_episodes
  - epsilon schedule (eps_start, eps_end, eps_decay)
- reward:
  - beta_coverage, lambda_cost
  - cost_mode, distance_cost_scale
- evaluation:
  - regime in {spread, dynamic, spatial}

D) Regime-specific runtime inputs
- spread: typically one frozen snapshot start.
- dynamic: sequence of snapshots; temporal evolution matters.
- spatial: spatial utility terms (coverage/cost) active.

E) Transfer protocol inputs
For each transfer target cell, the run receives:
- target size N (e.g., 300 or 500)
- target budget k (5,10,15 in corrected sweep)
- regime-specific tuned artifacts from source subset
- adaptation iterations and start snapshot

Important correction applied:
- scripts/evaluate_transfer_homogeneous.py now preserves target budget from transfer config.
- This prevents tuned artifact files from forcing an old source budget (e.g., 3).
- So the k=5,10,15 transfer comparison is now valid.

B) Corrected transfer budget sweep (N=300,500 and k=5,10,15)
Source tables generated from corrected artifacts:
- artifacts/paper_minimal/tables/table_transfer_budget_sweep.md
- artifacts/paper_minimal/tables/table_transfer_budget_sweep.tex

Key patterns by regime:

1. Spread
- Best RL mode beats baseline in only 1 out of 6 cells.
- Mean gap vs baseline (higher is better; 0 means tie):
  - Zero-shot:  -14.474
  - Warm-start: -10.313
  - Scratch:    -23.677
- Interpretation:
  Warm-start is usually the least-bad RL mode in spread transfer,
  but spread baselines remain strong overall, especially at higher k.

2. Dynamic
- Best RL mode beats baseline in 0 out of 6 cells.
- Mean gap vs baseline:
  - Zero-shot:  -15.839
  - Warm-start: -15.475
  - Scratch:    -14.402
- Interpretation:
  All RL transfer modes are close to each other but below dynamic baseline.
  Dynamic transfer remains the hardest regime in this run.

3. Spatial
- Best RL mode beats baseline in 3 out of 6 cells.
- Mean gap vs baseline:
  - Zero-shot:  -22.485
  - Warm-start: -18.908
  - Scratch:    -2.129
- Interpretation:
  Spatial has the most competitive RL transfer behavior.
  Scratch adaptation often performs best among RL modes, and some cells
  surpass baseline (example: N=500, k=10 zero-shot gap +3.984).

Across all 18 transfer cells (2 targets x 3 budgets x 3 regimes):
- Best RL mode winner counts:
  - Scratch: 10 cells
  - Warm-start: 4 cells
  - Zero-shot: 4 cells
Interpretation:
- No single transfer mode dominates all conditions.
- Scratch is most frequently best among RL modes in this experiment.
